# Trees & Graphs — AI నిర్ణయాలు ఎలా తీసుకుంటుంది

Trees మరియు graphs AI లో **ప్రతిచోటా** ఉన్నాయి: decision trees డేటాను వర్గీకరిస్తాయి, knowledge graphs సంబంధాలను నిల్వ చేస్తాయి, మరియు neural networks directed acyclic graphs. Traversal లో నైపుణ్యం సాధించడం తప్పనిసరి.

---

## Binary Tree Traversals

ప్రతి tree సమస్య దీనికి దిగుతుంది: **మీరు ప్రతి node ను ఎలా సందర్శిస్తారు?**

<Illustration src="/images/lessons/craft/tree-traversal.svg" alt="BFS level-order vs DFS pre-order traversal చూపించే binary tree, సంఖ్యాత్మక సందర్శన క్రమంతో" caption="BFS స్థాయి వారీగా సందర్శిస్తుంది (queue ఉపయోగించి). DFS మొదట లోతుగా వెళ్తుంది (stack/recursion ఉపయోగించి). ఒకే tree, వేరువేరు సందర్శన క్రమాలు." />

### BFS — Level Order (Queue)

```python
from collections import deque

def level_order(root):
    if not root:
        return []
    result, queue = [], deque([root])

    while queue:
        level = []
        for _ in range(len(queue)):   # process one level
            node = queue.popleft()
            level.append(node.val)
            if node.left:  queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)

    return result
```

### DFS — మూడు రకాలు (Stack/Recursion)

```python
def preorder(node):    # Root -> Left -> Right
    if not node: return
    visit(node)
    preorder(node.left)
    preorder(node.right)

def inorder(node):     # Left -> Root -> Right (sorted for BST!)
    if not node: return
    inorder(node.left)
    visit(node)
    inorder(node.right)

def postorder(node):   # Left -> Right -> Root (delete safely)
    if not node: return
    postorder(node.left)
    postorder(node.right)
    visit(node)
```

<Callout type="info">
**ఏది ఎప్పుడు వాడాలి?** BFS కనిష్ట మార్గం / స్థాయి-వారీ ప్రాసెసింగ్ కోసం. DFS మార్గం కనుగొనడం, లోతు గణన, లేదా తల్లిదండ్రులకు ముందు పిల్లలను ప్రాసెస్ చేయాల్సినప్పుడు (postorder) కోసం.
</Callout>

### క్లాసిక్ tree సమస్యలు

| సమస్య | Traversal | ముఖ్య అంతర్దృష్టి |
|--------|-----------|-------------------|
| గరిష్ట లోతు | DFS | `1 + max(depth(left), depth(right))` |
| Level order traversal | BFS | స్థాయి-పరిమాణ బ్యాచింగ్ తో Queue |
| BST ధృవీకరించండి | Inorder DFS | విలువలు ఖచ్చితంగా పెరుగుతూ ఉండాలి |
| అత్యల్ప ఉమ్మడి పూర్వీకుడు | DFS | Recurse చేయండి, ఎడమ/కుడి రిటర్న్‌లు తనిఖీ చేయండి |
| Serialize/Deserialize | BFS లేదా Preorder | Nulls ను మార్కర్లుగా ఎన్‌కోడ్ చేయండి |

## Graph Traversals

Graphs అనేవి "cycles లేవు" అనే హామీ లేని trees. ముఖ్య అదనపు భాగం: అనంత loops నివారించడానికి **visited set**.

<Illustration src="/images/lessons/craft/graph-bfs-dfs.svg" alt="9 nodes ఉన్న graph, BFS వేవ్‌ఫ్రంట్ విస్తరణ vs DFS లోతు-మొదట అన్వేషణ చూపిస్తోంది" caption="BFS విస్తరిస్తున్న తరంగ ముఖాలలో అన్వేషిస్తుంది (కనిష్ట దూరం హామీ). DFS వెనక్కి తిరగడానికి ముందు ఒక మార్గంలో లోతుగా వెళ్తుంది." />

### Graph BFS

```python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])

    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```

### Graph DFS

```python
def dfs(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)

    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
```

<ThinkAboutIt>
BFS భారం లేని graph లో కనిష్ట మార్గాన్ని కనుగొనడానికి ఎందుకు హామీ ఇస్తుంది? ఎందుకంటే ఇది దూరం d+1 లోని ఏ node కంటే ముందు దూరం d లోని అన్ని nodes ను అన్వేషిస్తుంది. DFS కి అలాంటి హామీ లేదు — ఇది మొదట పొడవైన మార్గాన్ని కనుగొనవచ్చు.
</ThinkAboutIt>

### క్లాసిక్ graph సమస్యలు

| సమస్య | అల్గారిథం | సంక్లిష్టత |
|--------|-----------|-----------|
| కనిష్ట మార్గం (భారం లేని) | BFS | O(V + E) |
| Cycle గుర్తింపు (దిశ లేని) | DFS parent tracking తో | O(V + E) |
| Cycle గుర్తింపు (దిశ ఉన్న) | DFS 3-రంగు గుర్తింపుతో | O(V + E) |
| దీవుల సంఖ్య | BFS/DFS flood fill | O(అడ్డు వరుసలు * నిలువు వరుసలు) |
| Topological sort | DFS postorder రివర్స్ | O(V + E) |
| అనుసంధాన భాగాలు | BFS/DFS ప్రతి సందర్శించని దాని నుండి | O(V + E) |

## AI కనెక్షన్: ML లో Trees & Graphs

<FunFact>
Trees మరియు graphs కేవలం డేటా నిర్మాణాలు మాత్రమే కాదు — ఇవి ఆధునిక AI యొక్క వెన్నెముక:

- **Decision Trees & Random Forests:** ప్రతి అంచనా ఒక binary tree లో మూలం నుండి ఆకు వరకు నడుస్తుంది. Traversal క్రమం = feature ప్రాముఖ్యత ర్యాంకింగ్.
- **Knowledge Graphs:** Google యొక్క Knowledge Graph, Wikidata, మరియు ఎంటర్‌ప్రైజ్ ఆన్టాలజీలు ప్రశ్నలకు సమాధానం ఇవ్వడానికి BFS/DFS తో traverse చేయబడే భారీ directed graphs.
- **Graph Neural Networks (GNNs):** GNNs లో message passing అక్షరాలా BFS — ప్రతి node తన k-hop పొరుగు నుండి features ను సమీకరిస్తుంది.
- **Abstract Syntax Trees:** కోడ్ రాసే LLMs (Copilot, Codex) ASTs ను parse మరియు generate చేస్తాయి — కోడ్‌ను సూచించే tree నిర్మాణాలు.
- **Neural Network DAGs:** ఒక neural network *ఒక* directed acyclic graph. Backpropagation అనేది computation graph ద్వారా రివర్స్ postorder DFS.
</FunFact>

## శీఘ్ర సూచన

| నిర్మాణం | Traversal | డేటా నిర్మాణం | స్థలం | దీనికి ఉత్తమం |
|---------|-----------|-------------|-------|-------------|
| Tree BFS | Level order | Queue | O(w) | స్థాయి ప్రాసెసింగ్, కనిష్ట మార్గం |
| Tree DFS | Pre/In/Post | Stack/Recursion | O(h) | మార్గ సమస్యలు, BST ధృవీకరణ |
| Graph BFS | వేవ్‌ఫ్రంట్ | Queue + Visited | O(V) | కనిష్ట మార్గం, అనుసంధాన భాగాలు |
| Graph DFS | లోతు మొదట | Stack + Visited | O(V) | Cycle గుర్తింపు, topological sort |

*w = గరిష్ట వెడల్పు, h = ఎత్తు, V = శీర్షాలు*
