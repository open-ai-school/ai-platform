# Two Pointers & Sliding Window

ఈ రెండు నమూనాలు చాలా మంది O(n²) లో brute-force చేసే సమస్యలను **O(n)** లో పరిష్కరించడానికి మిమ్మల్ని అనుమతిస్తాయి. ఇవి ఒక ప్రధాన ఆలోచనను పంచుకుంటాయి: *డేటాను మళ్ళీ స్కాన్ చేయడానికి బదులు దానిలో ఒక తెలివైన విండోను నిర్వహించండి.*

---

## నమూనా 1: Two Pointers

ఒక **సార్ట్ చేయబడిన** array యొక్క ప్రతి చివరన ఒక pointer ఉంచండి. ఒక షరతు ఆధారంగా వాటిని లోపలికి కదపండి.

<Illustration src="/images/lessons/craft/two-pointers.svg" alt="లక్ష్య మొత్తాన్ని కనుగొనడానికి సార్ట్ చేయబడిన array పై L మరియు R రెండు pointers కలుస్తున్నాయి" caption="కలుస్తున్న two pointers: మొత్తాన్ని లక్ష్యంతో పోల్చండి, చాలా చిన్నదైతే L ను కుడికి, చాలా పెద్దదైతే R ను ఎడమకు కదపండి." />

### ఎప్పుడు వాడాలి

- **సార్ట్ చేయబడిన array** + జత/షరతు కోసం వెతకడం
- O(n²) జత శోధనలను O(n) కు తగ్గించడం
- విభజన (Dutch National Flag)

### అల్గారిథం

```python
def two_sum_sorted(arr, target):
    left, right = 0, len(arr) - 1

    while left < right:
        current_sum = arr[left] + arr[right]
        if current_sum == target:
            return [left, right]
        elif current_sum < target:
            left += 1       # need bigger sum
        else:
            right -= 1      # need smaller sum

    return [-1, -1]
```

<Callout type="info">
**ముఖ్య అపరివర్తనీయం:** `L` కు ఎడమ వైపు ఉన్నవన్నీ చాలా చిన్నవి. `R` కు కుడి వైపు ఉన్నవన్నీ చాలా పెద్దవి. సమాధానం *వాటి మధ్యలో* తప్పనిసరిగా ఉండాలి.
</Callout>

## నమూనా 2: Sliding Window

Array పై జారే **విండో** [left, right] ను నిర్వహించండి. కుడికి విస్తరించండి, ఎడమకు కుదించండి — విండోను ఎప్పుడూ మళ్ళీ స్కాన్ చేయవద్దు.

<Illustration src="/images/lessons/craft/sliding-window.svg" alt="నడుస్తున్న మొత్తాన్ని ట్రాక్ చేస్తూ array పై కదులుతున్న k పరిమాణపు sliding window" caption="స్థిర-పరిమాణ విండో కుడికి జారుతుంది: బయటకు వెళ్ళే ఎలిమెంట్‌ను తీసివేయండి, లోపలికి వచ్చే ఎలిమెంట్‌ను జోడించండి. మొత్తం O(n)." />

### రెండు రకాలు

**స్థిర-పరిమాణ విండో (పరిమాణం k):**

```python
def max_sum_subarray(arr, k):
    window_sum = sum(arr[:k])
    max_sum = window_sum

    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i - k]  # slide: add new, remove old
        max_sum = max(max_sum, window_sum)

    return max_sum
```

**మారుతున్న-పరిమాణ విండో (విస్తరించు/కుదించు):**

```python
def longest_substring_no_repeat(s):
    seen = set()
    left = max_len = 0

    for right in range(len(s)):
        while s[right] in seen:
            seen.remove(s[left])
            left += 1              # shrink window
        seen.add(s[right])
        max_len = max(max_len, right - left + 1)

    return max_len
```

<ThinkAboutIt>
మారుతున్న విండో ఇప్పటికీ O(n) లో ఎందుకు నడుస్తుంది? ప్రతి ఎలిమెంట్ set కు ఒకసారి జోడించబడుతుంది మరియు గరిష్టంగా ఒకసారి తీసివేయబడుతుంది — మొత్తం 2n ఆపరేషన్లు.
</ThinkAboutIt>

### క్లాసిక్ సమస్యలు

| సమస్య | విండో రకం | ముఖ్య అంతర్దృష్టి |
|--------|-----------|-------------------|
| k పరిమాణపు ఉప-శ్రేణి గరిష్ట మొత్తం | స్థిర | జారించి మొత్తాన్ని ట్రాక్ చేయండి |
| పునరావృతం లేని పొడవైన substring | మారుతున్న | కుడికి విస్తరించండి, నకిలీపై ఎడమకు కుదించండి |
| కనిష్ట విండో substring | మారుతున్న | అన్ని అక్షరాలు కనుగొనబడినప్పుడు కుదించండి |
| గరిష్ట వరుస ఒకటులు (k flips) | మారుతున్న | flips k ను మించినప్పుడు కుదించండి |

## AI కనెక్షన్: Streaming డేటా ప్రాసెసింగ్

<FunFact>
ఇవి కేవలం ఇంటర్వ్యూ నమూనాలు మాత్రమే కాదు — ప్రొడక్షన్ ML వ్యవస్థలు ఇలాగే పనిచేస్తాయి:

- **ML లో Sliding window:** టైమ్-సీరీస్ డేటాపై feature నిష్కర్షణ (ఉదా., "గడిచిన 5 నిమిషాలలో సగటు CPU వాడకం") సరిగ్గా అదే స్థిర-విండో టెక్నిక్‌ను ఉపయోగిస్తుంది.
- **శోధనలో Two pointers:** పంపిణీ చేయబడిన సార్టింగ్ (MapReduce) లో merge ఆపరేషన్లు సార్ట్ చేయబడిన విభాగాలపై two-pointer merging ఉపయోగిస్తాయి.
- **Streaming ఎనలిటిక్స్:** Apache Kafka మరియు Flink sliding మరియు tumbling windows ఉపయోగించి అనంత డేటా స్ట్రీమ్‌లను ప్రాసెస్ చేస్తాయి — అదే O(n) సింగిల్-పాస్ లాజిక్.
- **Attention మెకానిజమ్‌లు:** Transformers లో sliding window attention (ఉదా., Longformer) స్థిర-పరిమాణ విండో ఉపయోగించి ఏ tokens ఒకదానికొకటి attend అవుతాయో పరిమితం చేస్తుంది.
</FunFact>
