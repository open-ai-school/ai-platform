# Two Pointers & Sliding Window

Deze twee patronen laten je problemen in **O(n)** oplossen die de meeste mensen brute-forcen in O(n²). Ze delen een kernidee: *onderhoud een slim venster op de data in plaats van opnieuw te scannen.*

---

## Patroon 1: Two Pointers

Plaats één pointer aan elk uiteinde van een **gesorteerde** array. Beweeg ze naar binnen op basis van een voorwaarde.

<Illustration src="/images/lessons/craft/two-pointers.svg" alt="Twee pointers L en R die convergeren op een gesorteerde array om een doelsom te vinden" caption="Convergerende two pointers: vergelijk de som met het doel, beweeg L naar rechts als te klein, R naar links als te groot." />

### Wanneer gebruiken

- **Gesorteerde array** + zoeken naar een paar/voorwaarde
- O(n²) paarzoekopdrachten reduceren tot O(n)
- Partitionering (Dutch National Flag)

### Het algoritme

```python
def two_sum_sorted(arr, target):
    left, right = 0, len(arr) - 1

    while left < right:
        current_sum = arr[left] + arr[right]
        if current_sum == target:
            return [left, right]
        elif current_sum < target:
            left += 1       # need bigger sum
        else:
            right -= 1      # need smaller sum

    return [-1, -1]
```

<Callout type="info">
**Belangrijkste invariant:** Alles links van `L` is te klein. Alles rechts van `R` is te groot. Het antwoord *moet* ertussen liggen.
</Callout>

### Klassieke problemen

| Probleem | Truc | Complexiteit |
|----------|------|-------------|
| Two Sum (gesorteerd) | Convergeren naar doel | O(n) |
| Container with Most Water | Kortste muur naar binnen | O(n) |
| 3Sum | Eén fixeren, two-pointer de rest | O(n²) |
| Remove Duplicates (gesorteerd) | Langzame/snelle pointer | O(n) |

## Patroon 2: Sliding Window

Onderhoud een **venster** [left, right] dat over de array schuift. Uitbreiden naar rechts, krimpen naar links — nooit het venster opnieuw scannen.

<Illustration src="/images/lessons/craft/sliding-window.svg" alt="Een glijdend venster van grootte k dat over een array beweegt en de lopende som bijhoudt" caption="Venster met vaste grootte schuift naar rechts: vertrekkend element aftrekken, binnenkomend element optellen. O(n) totaal." />

### Twee varianten

**Venster met vaste grootte (grootte k):**

```python
def max_sum_subarray(arr, k):
    window_sum = sum(arr[:k])
    max_sum = window_sum

    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i - k]  # slide: add new, remove old
        max_sum = max(max_sum, window_sum)

    return max_sum
```

**Venster met variabele grootte (uitbreiden/krimpen):**

```python
def longest_substring_no_repeat(s):
    seen = set()
    left = max_len = 0

    for right in range(len(s)):
        while s[right] in seen:
            seen.remove(s[left])
            left += 1              # shrink window
        seen.add(s[right])
        max_len = max(max_len, right - left + 1)

    return max_len
```

<ThinkAboutIt>
Waarom draait het variabele venster nog steeds in O(n)? Elk element wordt één keer aan de set toegevoegd en maximaal één keer verwijderd — dat zijn 2n bewerkingen in totaal.
</ThinkAboutIt>

### Klassieke problemen

| Probleem | Venstertype | Kernidee |
|----------|-------------|----------|
| Max som subarray van grootte k | Vast | Schuiven en som bijhouden |
| Langste substring zonder herhalingen | Variabel | Rechts uitbreiden, links krimpen bij duplicaat |
| Minimale venster-substring | Variabel | Krimpen wanneer alle tekens gevonden |
| Max opeenvolgende enen (k flips) | Variabel | Krimpen wanneer flips k overschrijden |

## AI-connectie: Streaming dataverwerking

<FunFact>
Dit zijn niet alleen sollicitatiepatronen — zo werken productie-ML-systemen:

- **Sliding window in ML:** Feature-extractie over tijdreeksdata (bijv. "gemiddeld CPU-gebruik over de laatste 5 minuten") gebruikt exact dezelfde techniek met vast venster.
- **Two pointers in zoeken:** Merge-operaties in gedistribueerd sorteren (MapReduce) gebruiken two-pointer-merging op gesorteerde partities.
- **Streaming-analyse:** Apache Kafka en Flink verwerken oneindige datastromen met glijdende en kantelende vensters — dezelfde O(n) single-pass logica.
- **Aandachtsmechanismen:** Sliding window-aandacht in transformers (bijv. Longformer) beperkt welke tokens op elkaar letten met een venster van vaste grootte.
</FunFact>
