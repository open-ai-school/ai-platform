# Bomen & Grafen — Hoe AI beslissingen neemt

Bomen en grafen zijn **overal** in AI: beslissingsbomen classificeren data, kennisgrafen slaan relaties op, en neurale netwerken zijn gerichte acyclische grafen. Het beheersen van traversals is ononderhandelbaar.

---

## Binaire boom-traversals

Elk boomprobleem komt neer op: **hoe bezoek je elk knooppunt?**

<Illustration src="/images/lessons/craft/tree-traversal.svg" alt="Binaire boom met BFS level-order versus DFS pre-order traversal met genummerde bezoekvolgorde" caption="BFS bezoekt niveau voor niveau (met een queue). DFS duikt eerst diep (met een stack/recursie). Dezelfde boom, verschillende bezoekvolgorden." />

### BFS — Level Order (Queue)

```python
from collections import deque

def level_order(root):
    if not root:
        return []
    result, queue = [], deque([root])

    while queue:
        level = []
        for _ in range(len(queue)):   # process one level
            node = queue.popleft()
            level.append(node.val)
            if node.left:  queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)

    return result
```

### DFS — Drie varianten (Stack/Recursie)

```python
def preorder(node):    # Root -> Left -> Right
    if not node: return
    visit(node)
    preorder(node.left)
    preorder(node.right)

def inorder(node):     # Left -> Root -> Right (sorted for BST!)
    if not node: return
    inorder(node.left)
    visit(node)
    inorder(node.right)

def postorder(node):   # Left -> Right -> Root (delete safely)
    if not node: return
    postorder(node.left)
    postorder(node.right)
    visit(node)
```

<Callout type="info">
**Wanneer welke gebruiken?** BFS voor het kortste pad / niveau-voor-niveau verwerking. DFS voor padzoeken, diepteberekening, of wanneer je kinderen vóór ouders moet verwerken (postorder).
</Callout>

### Klassieke boomproblemen

| Probleem | Traversal | Kernidee |
|----------|-----------|----------|
| Maximale diepte | DFS | `1 + max(depth(left), depth(right))` |
| Level order traversal | BFS | Queue met level-grootte batching |
| BST valideren | Inorder DFS | Waarden moeten strikt toenemen |
| Laagste gemeenschappelijke voorouder | DFS | Recursie, controleer links/rechts returns |
| Serialiseren/Deserialiseren | BFS of Preorder | Nulls coderen als markers |

## Graaf-traversals

Grafen zijn bomen zonder de garantie van "geen cycli". De belangrijkste toevoeging: **een visited-set** om oneindige lussen te vermijden.

<Illustration src="/images/lessons/craft/graph-bfs-dfs.svg" alt="Graaf met 9 knooppunten die BFS-golffrontexpansie versus DFS diepte-eerst exploratie toont" caption="BFS verkent in uitbreidende golffronten (kortste afstand gegarandeerd). DFS duikt één pad af voordat het terugkeert." />

### Graph BFS

```python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])

    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```

### Graph DFS

```python
def dfs(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)

    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
```

<ThinkAboutIt>
Waarom is BFS gegarandeerd het kortste pad te vinden in een ongewogen graaf? Omdat het alle knooppunten op afstand d verkent vóór elk knooppunt op afstand d+1. DFS biedt deze garantie niet — het kan eerst een langer pad vinden.
</ThinkAboutIt>

### Klassieke graafproblemen

| Probleem | Algoritme | Complexiteit |
|----------|-----------|-------------|
| Kortste pad (ongewogen) | BFS | O(V + E) |
| Cyclusdetectie (ongericht) | DFS met ouder-tracking | O(V + E) |
| Cyclusdetectie (gericht) | DFS met 3-kleurenmarkering | O(V + E) |
| Aantal eilanden | BFS/DFS flood fill | O(rijen * kolommen) |
| Topologische sortering | DFS postorder omgekeerd | O(V + E) |
| Samenhangende componenten | BFS/DFS vanaf elk onbezocht | O(V + E) |

## AI-connectie: Bomen & Grafen in ML

<FunFact>
Bomen en grafen zijn niet alleen datastructuren — ze vormen de ruggengraat van moderne AI:

- **Beslissingsbomen & Random Forests:** Elke voorspelling loopt door een binaire boom van wortel naar blad. Traversalvolgorde = rangschikking van feature-belang.
- **Kennisgrafen:** Googles Knowledge Graph, Wikidata en bedrijfsontologieën zijn enorme gerichte grafen die met BFS/DFS worden doorlopen om vragen te beantwoorden.
- **Graph Neural Networks (GNN's):** Berichtuitwisseling in GNN's is letterlijk BFS — elk knooppunt aggregeert features van zijn k-hop buurt.
- **Abstracte syntaxisbomen:** LLM's die code schrijven (Copilot, Codex) parseren en genereren AST's — boomstructuren die code representeren.
- **Neurale netwerk-DAG's:** Een neuraal netwerk *is* een gerichte acyclische graaf. Backpropagation is omgekeerde postorder DFS door de berekeningsgraaf.
</FunFact>

## Snelle referentie

| Structuur | Traversal | Datastructuur | Ruimte | Beste voor |
|-----------|-----------|---------------|--------|------------|
| Tree BFS | Level order | Queue | O(w) | Levelverwerking, kortste pad |
| Tree DFS | Pre/In/Post | Stack/Recursie | O(h) | Padproblemen, BST-validatie |
| Graph BFS | Golffront | Queue + Visited | O(V) | Kortste pad, samenhangende componenten |
| Graph DFS | Diepte eerst | Stack + Visited | O(V) | Cyclusdetectie, topologische sortering |

*w = max breedte, h = hoogte, V = hoekpunten*
