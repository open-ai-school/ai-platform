# Two Pointers & Sliding Window

Ces deux patterns vous permettent de résoudre en **O(n)** des problèmes que la plupart des gens résolvent en force brute en O(n²). Ils partagent une idée centrale : *maintenir une fenêtre intelligente sur les données au lieu de les re-parcourir.*

---

## Pattern 1 : Two Pointers

Placez un pointeur à chaque extrémité d'un tableau **trié**. Déplacez-les vers l'intérieur en fonction d'une condition.

<Illustration src="/images/lessons/craft/two-pointers.svg" alt="Deux pointeurs L et R convergeant sur un tableau trié pour trouver une somme cible" caption="Two pointers convergents : comparer la somme à la cible, déplacer L vers la droite si trop petit, R vers la gauche si trop grand." />

### Quand l'utiliser

- **Tableau trié** + recherche d'une paire/condition
- Réduire les recherches de paires O(n²) à O(n)
- Partitionnement (Dutch National Flag)

### L'algorithme

```python
def two_sum_sorted(arr, target):
    left, right = 0, len(arr) - 1

    while left < right:
        current_sum = arr[left] + arr[right]
        if current_sum == target:
            return [left, right]
        elif current_sum < target:
            left += 1       # need bigger sum
        else:
            right -= 1      # need smaller sum

    return [-1, -1]
```

<Callout type="info">
**Invariant clé :** Tout ce qui est à gauche de `L` est trop petit. Tout ce qui est à droite de `R` est trop grand. La réponse *doit* se trouver entre les deux.
</Callout>

### Problèmes classiques

| Problème | Astuce | Complexité |
|----------|--------|------------|
| Two Sum (trié) | Converger vers la cible | O(n) |
| Container with Most Water | Déplacer le mur le plus court vers l'intérieur | O(n) |
| 3Sum | Fixer un élément, two-pointer le reste | O(n²) |
| Remove Duplicates (trié) | Pointeur lent/rapide | O(n) |

## Pattern 2 : Sliding Window

Maintenir une **fenêtre** [left, right] qui glisse sur le tableau. Étendre à droite, rétrécir à gauche — ne jamais re-parcourir la fenêtre.

<Illustration src="/images/lessons/craft/sliding-window.svg" alt="Une fenêtre glissante de taille k se déplaçant sur un tableau en suivant la somme courante" caption="Fenêtre de taille fixe glissant vers la droite : soustraire l'élément sortant, ajouter l'élément entrant. O(n) au total." />

### Deux variantes

**Fenêtre de taille fixe (taille k) :**

```python
def max_sum_subarray(arr, k):
    window_sum = sum(arr[:k])
    max_sum = window_sum

    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i - k]  # slide: add new, remove old
        max_sum = max(max_sum, window_sum)

    return max_sum
```

**Fenêtre de taille variable (étendre/rétrécir) :**

```python
def longest_substring_no_repeat(s):
    seen = set()
    left = max_len = 0

    for right in range(len(s)):
        while s[right] in seen:
            seen.remove(s[left])
            left += 1              # shrink window
        seen.add(s[right])
        max_len = max(max_len, right - left + 1)

    return max_len
```

<ThinkAboutIt>
Pourquoi la fenêtre variable s'exécute-t-elle toujours en O(n) ? Chaque élément est ajouté à l'ensemble une fois et retiré au plus une fois — soit 2n opérations au total.
</ThinkAboutIt>

### Problèmes classiques

| Problème | Type de fenêtre | Idée clé |
|----------|----------------|----------|
| Somme max d'un sous-tableau de taille k | Fixe | Glisser et suivre la somme |
| Plus longue sous-chaîne sans répétition | Variable | Étendre à droite, rétrécir à gauche en cas de doublon |
| Sous-chaîne fenêtre minimale | Variable | Rétrécir quand tous les caractères sont trouvés |
| Max uns consécutifs (k inversions) | Variable | Rétrécir quand les inversions dépassent k |

## Connexion IA : Traitement de données en streaming

<FunFact>
Ce ne sont pas que des patterns d'entretien — c'est ainsi que les systèmes ML en production fonctionnent :

- **Sliding window en ML :** L'extraction de caractéristiques sur des séries temporelles (par ex., « utilisation moyenne du CPU sur les 5 dernières minutes ») utilise exactement la même technique de fenêtre fixe.
- **Two pointers dans la recherche :** Les opérations de fusion dans le tri distribué (MapReduce) utilisent la fusion two-pointer sur des partitions triées.
- **Analytique en streaming :** Apache Kafka et Flink traitent des flux de données infinis en utilisant des fenêtres glissantes et des fenêtres à bascule — la même logique O(n) en un seul passage.
- **Mécanismes d'attention :** L'attention à fenêtre glissante dans les transformers (par ex., Longformer) limite quels tokens interagissent entre eux en utilisant une fenêtre de taille fixe.
</FunFact>

| Pattern | Quand | Temps | Espace |
|---------|-------|-------|--------|
| Two Pointers (convergent) | Tableau trié, trouver une paire | O(n) | O(1) |
| Two Pointers (rapide/lent) | Cycle de liste chaînée, partitionnement | O(n) | O(1) |
| Sliding Window fixe | Sous-tableau de taille exacte k | O(n) | O(1) |
| Sliding Window variable | Plus long/court sous-tableau avec condition | O(n) | O(k) |
