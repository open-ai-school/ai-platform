# Strings & Pattern Matching

Chaque fois que vous tapez une requête de recherche, envoyez un message ou posez une question à une IA — des strings sont traitées. Comprendre leur fonctionnement interne débloque à la fois la réussite en entretien et l'intuition de l'IA.

---

## Les strings sous le capot

Un string est simplement un **array de caractères** avec des règles spéciales.

<Illustration src="/images/lessons/craft/strings-patterns.svg" alt="String as character array, palindrome two-pointer check, and sliding window technique" caption="Les strings sont des arrays de caractères (en haut à gauche). Les palindromes sont vérifiés avec deux pointeurs partant des deux extrémités (en haut à droite). La fenêtre glissante trouve les sous-chaînes optimales (en bas)." />

**Faits clés :**

- **Immuables** dans la plupart des langages (Java, Python, Go) — chaque « modification » crée un nouveau string
- **Indexés** — accès à n'importe quel caractère en O(1)
- **Encodés** — UTF-8, UTF-16, ASCII affectent la correspondance entre caractères et octets

<Callout type="info">
En Python, `s = "hello"` puis `s[0] = "H"` provoque une erreur. Les strings sont **immuables**. Utilisez `s = "H" + s[1:]` ou `list(s)` pour les modifier.
</Callout>

## Pattern 1 : Inversion de string

> Inverser un string sur place à l'aide de deux pointeurs.

**Démonstration visuelle** — `"HELLO"` :

| Étape | Ptr G | Ptr D | Action | Résultat |
|-------|-------|-------|--------|----------|
| 1 | H (0) | O (4) | Échange | **O**ELL**H** |
| 2 | E (1) | L (3) | Échange | O**L**L**E**H |
| 3 | L=L | — | Terminé | OLLEH |

```python
def reverse_string(s: list[str]) -> None:
    left, right = 0, len(s) - 1
    while left < right:
        s[left], s[right] = s[right], s[left]
        left += 1
        right -= 1
```

**Temps : O(n) | Espace : O(1)** — aucune mémoire supplémentaire nécessaire.

## Pattern 2 : Vérification de palindrome

> Le string est-il identique à l'endroit et à l'envers ?

**Approche à deux pointeurs** — comparer depuis les deux extrémités, en se rapprochant :

1. Définir `left = 0`, `right = len - 1`
2. Si `s[left] != s[right]` → **ce n'est pas un palindrome**
3. Déplacer les deux pointeurs vers l'intérieur
4. S'ils se rencontrent → **palindrome confirmé**

```python
def is_palindrome(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True
```

<ThinkAboutIt>
Comment géreriez-vous « A man, a plan, a canal: Panama » ? Vous devez **ignorer les caractères non alphanumériques** et **ne pas tenir compte de la casse**. Ajoutez `while left &lt; right and not s[left].isalnum(): left += 1` avant chaque comparaison.
</ThinkAboutIt>

## Pattern 3 : Détection d'anagrammes

> Deux strings sont-ils des anagrammes ? (Mêmes caractères, ordre différent)

**Approche hash map** — compter les fréquences de caractères :

```python
def is_anagram(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False
    count = {}
    for c in s:
        count[c] = count.get(c, 0) + 1
    for c in t:
        count[c] = count.get(c, 0) - 1
        if count[c] < 0:
            return False
    return True
```

| | "listen" | "silent" |
|---|----------|----------|
| l | 1 | 1 |
| i | 1 | 1 |
| s | 1 | 1 |
| t | 1 | 1 |
| e | 1 | 1 |
| n | 1 | 1 |

Tous les compteurs à zéro → **anagramme confirmé**.

## Pattern 4 : Fenêtre glissante

> Trouver la plus longue sous-chaîne sans caractères répétés.

**La technique :** Maintenir une « fenêtre » de caractères valides. Étendre à droite, réduire à gauche quand un doublon est trouvé.

**Démonstration** avec `"ABCABCBB"` :

| Étape | Fenêtre | Vus | Longueur |
|-------|---------|-----|----------|
| 1 | **A** | A | 1 |
| 2 | **AB** | A, B | 2 |
| 3 | **ABC** | A, B, C | 3 |
| 4 | A est un doublon → réduire → **BCA** | B, C, A | 3 |
| 5 | B est un doublon → réduire → **CAB** | C, A, B | 3 |

**Réponse : 3** (n'importe lequel parmi "ABC", "BCA" ou "CAB")

```python
def length_of_longest_substring(s: str) -> int:
    seen = {}
    left = max_len = 0
    for right, char in enumerate(s):
        if char in seen and seen[char] >= left:
            left = seen[char] + 1
        seen[char] = right
        max_len = max(max_len, right - left + 1)
    return max_len
```

<Callout type="info">
Le pattern de fenêtre glissante résout **des dizaines** de problèmes de sous-chaînes. L'idée clé : si l'extension de la fenêtre viole une condition, réduisez depuis la gauche jusqu'à ce que la condition soit de nouveau respectée.
</Callout>

## Le lien avec l'IA : la tokenisation

Avant qu'un modèle NLP (GPT, BERT, LLaMA) ne traite du texte, il **tokenise** — découpe le texte en morceaux de sous-mots en utilisant des techniques basées sur les patterns ci-dessus.

**Comment fonctionne le BPE (Byte Pair Encoding) :**

1. Commencer avec des caractères individuels : `["h", "e", "l", "l", "o"]`
2. Compter les paires adjacentes — paire la plus fréquente : `("l", "l")`
3. Fusionner : `["h", "e", "ll", "o"]`
4. Répéter jusqu'à atteindre la taille de vocabulaire souhaitée

**Cela utilise :**

- **Le comptage de fréquences** (hash maps) pour trouver les paires communes
- **La fenêtre glissante** pour parcourir les caractères adjacents
- **La manipulation de strings** pour fusionner les tokens

<FunFact>
Le tokenizer de GPT-4 possède environ 100 000 tokens. Le mot « indescribable » pourrait devenir ["in", "desc", "rib", "able"] — 4 tokens. Les mots courants comme « the » sont un seul token. C'est pourquoi les modèles d'IA « pensent » en tokens, pas en mots !
</FunFact>

## Résumé des complexités

| Pattern | Temps | Espace | Technique clé |
|---------|-------|--------|---------------|
| Inversion de string | O(n) | O(1) | Deux pointeurs |
| Vérification de palindrome | O(n) | O(1) | Deux pointeurs |
| Détection d'anagrammes | O(n) | O(1)* | Comptage par hash map |
| Fenêtre glissante | O(n) | O(k) | Hash set + deux pointeurs |

*O(1) car l'ensemble de caractères est fixe (26 lettres)

<ThinkAboutIt>
Pourquoi la fenêtre glissante est-elle en O(n) et non en O(n²) ? Parce que chaque caractère est ajouté et retiré de la fenêtre **au plus une fois**. Le pointeur gauche ne recule jamais — donc le travail total sur toutes les étapes est linéaire.
</ThinkAboutIt>
