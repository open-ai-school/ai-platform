# Arrays & Hashing

ప్రతి AI వ్యవస్థ — recommendation engines నుండి search indexes వరకు — రెండు ప్రాథమిక అంశాలపై నిర్మించబడింది: **arrays** మరియు **hash maps**. వీటిలో నైపుణ్యం సాధిస్తే పునాదిలో నైపుణ్యం సాధించినట్లే.

---

## Arrays ఎలా పని చేస్తాయి

ఒక array మూలకాలను **అనుసంధాన మెమరీ**లో నిల్వ చేస్తుంది — ప్రతి స్లాట్ తదుపరి దాని నుండి ఒక అడుగు దూరంలో ఉంటుంది.

<Illustration src="/images/lessons/craft/arrays-hashing.svg" alt="Array memory layout showing contiguous cells with index labels and hash map with key-to-bucket mapping" caption="Arrays డేటాను అనుసంధాన మెమరీలో నిల్వ చేస్తాయి (ఎడమ). Hash maps ఒక hash function ఉపయోగించి keys ను bucket positions కు మ్యాప్ చేస్తాయి (కుడి)." />

**ముఖ్య లక్షణాలు:**

- **O(1) యాక్సెస్** — ఏ index కైనా నేరుగా వెళ్ళండి
- **O(n) insert/delete** — మూలకాలను షిఫ్ట్ చేయడం ఖరీదైనది
- **Cache-friendly** — వరుస మెమరీ = వేగవంతమైన CPU reads
- **స్థిర పరిమాణం** (చాలా భాషల్లో) — సామర్థ్యం ముందుగానే తెలియాలి

<Callout type="info">
Arrays మెషిన్ లెర్నింగ్‌లో **tensors** కు వెన్నెముక. ఒక 2D array ఒక matrix, ఒక 3D array ఒక tensor — ప్రతి neural network యొక్క ప్రధాన డేటా నిర్మాణం.
</Callout>

## Hash Maps ఎలా పని చేస్తాయి

ఒక hash map ఒక **key** ను hash function ఉపయోగించి ఒక **index** గా మారుస్తుంది, తర్వాత ఆ bucket లో విలువను నిల్వ చేస్తుంది.

**ప్రక్రియ:**

1. **ఇన్‌పుట్:** `"apple"` (key)
2. **Hash:** `hash("apple") % size` → bucket `0`
3. **స్టోర్:** bucket `0` = `apple: 3`
4. **Lookup:** సగటున O(1) — నేరుగా bucket కు వెళ్ళండి

**ముఖ్య లక్షణాలు:**

- **సగటున O(1)** lookup, insert, delete
- **అధ్వాన్న పరిస్థితిలో O(n)** — అన్ని keys ఒకే bucket లో ఢీకొన్నప్పుడు
- **క్రమం లేనివి** — హామీ ఇచ్చిన iteration order లేదు
- **Keys hashable అయి ఉండాలి** — immutable types బాగా పని చేస్తాయి

<ThinkAboutIt>
Python లో list ను dictionary key గా ఎందుకు ఉపయోగించలేరు? ఎందుకంటే lists mutable — మీరు hashing తర్వాత list ను మార్చితే, hash value మారుతుంది కానీ bucket మారదు, దీని వల్ల lookup విఫలమవుతుంది.
</ThinkAboutIt>

## Pattern 1: Two Sum

> ఒక array మరియు target ఇచ్చినప్పుడు, target కు సమానమైన మొత్తం ఇచ్చే రెండు సంఖ్యలను కనుగొనండి.

**Brute force:** ప్రతి జత తనిఖీ చేయండి → O(n²)
**Hash map విధానం:** ప్రతి సంఖ్యకు, `target - num` ఉందా అని తనిఖీ చేయండి → **O(n)**

**దశల వారీగా** `nums = [2, 7, 11, 15]`, `target = 9` తో:

| దశ | ప్రస్తుతం | అవసరం (9 - ప్రస్తుతం) | Map లో ఉందా? | చర్య |
|-----|-----------|----------------------|-------------|------|
| 1 | 2 | 7 | లేదు | `2: 0` నిల్వ చేయండి |
| 2 | 7 | 2 | **ఉంది!** | `[0, 1]` తిరిగి ఇవ్వండి |

<Callout type="info">
ఈ pattern — "మీరు చూసింది నిల్వ చేయండి, మీకు కావలసింది తనిఖీ చేయండి" — **డజన్ల కొద్దీ** interview సమస్యల్లో కనిపిస్తుంది.
</Callout>

## Pattern 2: Frequency Counting

> ప్రతి మూలకం ఎన్ని సార్లు కనిపిస్తుందో లెక్కించండి.

```python
def frequency_count(nums):
    freq = {}
    for num in nums:
        freq[num] = freq.get(num, 0) + 1
    return freq

# [1, 2, 2, 3, 3, 3] => {1: 1, 2: 2, 3: 3}
```

**ఉపయోగం:**

- **Top K Frequent Elements** — లెక్కించి frequency ప్రకారం sort చేయండి
- **Valid Anagram** — character frequencies ను పోల్చండి
- **Group Anagrams** — ఒకే sorted characters ఉన్న పదాలను group చేయండి

## Pattern 3: Key ద్వారా Grouping

> Anagrams ను group చేయండి: `["eat","tea","tan","ate","nat","bat"]`

**వ్యూహం:** ప్రతి పదాన్ని sort చేయండి → sorted version ను hash key గా ఉపయోగించండి.

| అసలు | Sorted (Key) | Group |
|-------|-------------|-------|
| "eat" | "aet" | ["eat", "tea", "ate"] |
| "tan" | "ant" | ["tan", "nat"] |
| "bat" | "abt" | ["bat"] |

## AI సంబంధం: Feature Stores

ప్రొడక్షన్ ML వ్యవస్థల్లో, **feature stores** వాటి ప్రధానంగా hash maps ను ఉపయోగిస్తాయి:

- **User ID → feature vector** — రియల్-టైమ్ recommendations కోసం తక్షణ lookup
- **Product ID → embeddings** — recommendation engines లో similarity search
- **Cache layer** — ఖరీదైన features ను మళ్ళీ compute చేయకుండా నివారించండి

<FunFact>
Netflix యొక్క recommendation engine వినియోగదారులను కంటెంట్‌తో మ్యాచ్ చేయడానికి **సెకనుకు లక్షల hash map lookups** చేస్తుంది. ప్రతి వినియోగదారు చూసిన చరిత్ర `movie_id: rating` జతల hash map గా నిల్వ చేయబడుతుంది, ఇది model inference సమయంలో O(1) feature retrieval ను అనుమతిస్తుంది.
</FunFact>

## Complexity చీట్ షీట్

| ఆపరేషన్ | Array | Hash Map |
|----------|-------|----------|
| Index ద్వారా యాక్సెస్ | O(1) | — |
| విలువ ద్వారా వెతకడం | O(n) | O(1) సగటు |
| చివరన insert | O(1)* | O(1) సగటు |
| మొదట insert | O(n) | O(1) సగటు |
| Delete | O(n) | O(1) సగటు |
| Space | O(n) | O(n) |

*dynamic arrays కోసం amortised

<ThinkAboutIt>
మీరు hash map బదులు array ను ఎప్పుడు ఎంచుకుంటారు? మీకు **క్రమబద్ధమైన డేటా**, **cache performance**, లేదా **కనిష్ట memory overhead** అవసరమైనప్పుడు. Hash maps వేగం కోసం space ను మార్పిడి చేస్తాయి — ప్రతి entry కు hash, key మరియు pointer కోసం overhead ఉంటుంది.
</ThinkAboutIt>
