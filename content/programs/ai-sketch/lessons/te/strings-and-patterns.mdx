# Strings & Pattern Matching

మీరు search query టైప్ చేసినప్పుడు, మెసేజ్ పంపినప్పుడు, లేదా AI కి ప్రశ్న అడిగినప్పుడు — ప్రతిసారీ strings ప్రాసెస్ అవుతున్నాయి. అవి లోపల ఎలా పని చేస్తాయో అర్థం చేసుకోవడం interview లో విజయాన్ని మరియు AI పట్ల అవగాహనను అన్‌లాక్ చేస్తుంది.

---

## Strings లోపలి నిర్మాణం

ఒక string అనేది ప్రత్యేక నియమాలు ఉన్న ఒక **character array** మాత్రమే.

<Illustration src="/images/lessons/craft/strings-patterns.svg" alt="String as character array, palindrome two-pointer check, and sliding window technique" caption="Strings character arrays (ఎడమ పైన). Palindromes రెండు చివరల నుండి రెండు pointers తో తనిఖీ చేయబడతాయి (కుడి పైన). Sliding window ఉత్తమ substrings కనుగొంటుంది (కింద)." />

**ముఖ్య వాస్తవాలు:**

- **Immutable** చాలా భాషల్లో (Java, Python, Go) — ప్రతి "మార్పు" కొత్త string సృష్టిస్తుంది
- **Indexed** — ఏ character ను అయినా O(1) లో యాక్సెస్ చేయండి
- **Encoded** — UTF-8, UTF-16, ASCII characters bytes కు ఎలా map అవుతాయో ప్రభావితం చేస్తాయి

<Callout type="info">
Python లో, `s = "hello"` తర్వాత `s[0] = "H"` error ఇస్తుంది. Strings **immutable**. మార్చడానికి `s = "H" + s[1:]` లేదా `list(s)` ఉపయోగించండి.
</Callout>

## Pattern 1: String Reversal

> రెండు pointers ఉపయోగించి string ను in-place లో తిరగేయండి.

**విజువల్ ప్రదర్శన** — `"HELLO"`:

| దశ | L ptr | R ptr | చర్య | ఫలితం |
|-----|-------|-------|------|-------|
| 1 | H (0) | O (4) | Swap | **O**ELL**H** |
| 2 | E (1) | L (3) | Swap | O**L**L**E**H |
| 3 | L=L | — | పూర్తి | OLLEH |

```python
def reverse_string(s: list[str]) -> None:
    left, right = 0, len(s) - 1
    while left < right:
        s[left], s[right] = s[right], s[left]
        left += 1
        right -= 1
```

**సమయం: O(n) | Space: O(1)** — అదనపు మెమరీ అవసరం లేదు.

## Pattern 2: Palindrome తనిఖీ

> String ముందు మరియు వెనుక ఒకే విధంగా ఉందా?

**Two-pointer విధానం** — రెండు చివరల నుండి పోల్చండి, లోపలికి కదలండి:

1. `left = 0`, `right = len - 1` సెట్ చేయండి
2. `s[left] != s[right]` అయితే → **palindrome కాదు**
3. రెండు pointers ను లోపలికి కదపండి
4. అవి కలిస్తే → **palindrome నిర్ధారణ**

```python
def is_palindrome(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True
```

<ThinkAboutIt>
"A man, a plan, a canal: Panama" ను మీరు ఎలా handle చేస్తారు? మీరు **non-alphanumeric characters ను దాటాలి** మరియు **case ను విస్మరించాలి**. ప్రతి పోలికకు ముందు `while left &lt; right and not s[left].isalnum(): left += 1` జోడించండి.
</ThinkAboutIt>

## Pattern 3: Anagram Detection

> రెండు strings anagrams ఆ? (అదే characters, వేరే క్రమం)

**Hash map విధానం** — character frequencies లెక్కించండి:

```python
def is_anagram(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False
    count = {}
    for c in s:
        count[c] = count.get(c, 0) + 1
    for c in t:
        count[c] = count.get(c, 0) - 1
        if count[c] < 0:
            return False
    return True
```

| | "listen" | "silent" |
|---|----------|----------|
| l | 1 | 1 |
| i | 1 | 1 |
| s | 1 | 1 |
| t | 1 | 1 |
| e | 1 | 1 |
| n | 1 | 1 |

అన్ని counts సున్నా → **anagram నిర్ధారణ**.

## Pattern 4: Sliding Window

> పునరావృత characters లేని పొడవైన substring కనుగొనండి.

**టెక్నిక్:** చెల్లుబాటు అయ్యే characters యొక్క "window" ను నిర్వహించండి. కుడికి విస్తరించండి, duplicate కనిపించినప్పుడు ఎడమ నుండి కుదించండి.

**ప్రదర్శన** `"ABCABCBB"` తో:

| దశ | Window | చూసినవి | పొడవు |
|-----|--------|---------|-------|
| 1 | **A** | A | 1 |
| 2 | **AB** | A, B | 2 |
| 3 | **ABC** | A, B, C | 3 |
| 4 | A duplicate → కుదించు → **BCA** | B, C, A | 3 |
| 5 | B duplicate → కుదించు → **CAB** | C, A, B | 3 |

**సమాధానం: 3** ("ABC", "BCA", లేదా "CAB" లో ఏదైనా)

```python
def length_of_longest_substring(s: str) -> int:
    seen = {}
    left = max_len = 0
    for right, char in enumerate(s):
        if char in seen and seen[char] >= left:
            left = seen[char] + 1
        seen[char] = right
        max_len = max(max_len, right - left + 1)
    return max_len
```

<Callout type="info">
Sliding window pattern **డజన్ల కొద్దీ** substring సమస్యలను పరిష్కరిస్తుంది. ముఖ్య అంతర్దృష్టి: window విస్తరణ ఒక షరతును ఉల్లంఘిస్తే, షరతు మళ్ళీ నెరవేరే వరకు ఎడమ నుండి కుదించండి.
</Callout>

## AI సంబంధం: Tokenization

ఏదైనా NLP model (GPT, BERT, LLaMA) టెక్స్ట్ ప్రాసెస్ చేయడానికి ముందు, అది **tokenize** చేస్తుంది — పైన ఉన్న patterns పై ఆధారపడిన టెక్నిక్‌లను ఉపయోగించి టెక్స్ట్ ను sub-word ముక్కలుగా విభజిస్తుంది.

**BPE (Byte Pair Encoding) ఎలా పని చేస్తుంది:**

1. వ్యక్తిగత characters తో ప్రారంభించండి: `["h", "e", "l", "l", "o"]`
2. ప్రక్కనున్న pairs లెక్కించండి — అత్యంత తరచుగా వచ్చే pair: `("l", "l")`
3. Merge చేయండి: `["h", "e", "ll", "o"]`
4. కావలసిన vocabulary size వచ్చే వరకు పునరావృతం చేయండి

**ఇది ఉపయోగిస్తుంది:**

- సాధారణ pairs కనుగొనడానికి **frequency counting** (hash maps)
- ప్రక్కనున్న characters స్కాన్ చేయడానికి **sliding window**
- Tokens merge చేయడానికి **string manipulation**

<FunFact>
GPT-4 tokenizer లో సుమారు 1,00,000 tokens ఉన్నాయి. "indescribable" అనే పదం ["in", "desc", "rib", "able"] గా మారవచ్చు — 4 tokens. "the" వంటి సాధారణ పదాలు ఒకే token. అందుకే AI models పదాల్లో కాదు, tokens లో "ఆలోచిస్తాయి"!
</FunFact>

## Complexity సారాంశం

| Pattern | సమయం | Space | ముఖ్య టెక్నిక్ |
|---------|-------|-------|----------------|
| String reversal | O(n) | O(1) | Two pointers |
| Palindrome check | O(n) | O(1) | Two pointers |
| Anagram detection | O(n) | O(1)* | Hash map counting |
| Sliding window | O(n) | O(k) | Hash set + two pointers |

*O(1) ఎందుకంటే character set స్థిరం (26 అక్షరాలు)

<ThinkAboutIt>
Sliding window O(n) ఎందుకు మరియు O(n²) కాదు? ఎందుకంటే ప్రతి character window కు **గరిష్టంగా ఒక్కసారి** జోడించబడుతుంది మరియు తొలగించబడుతుంది. ఎడమ pointer ముందుకు మాత్రమే కదులుతుంది, ఎప్పుడూ వెనుకకు కాదు — కాబట్టి అన్ని దశల్లో మొత్తం పని linear.
</ThinkAboutIt>
