## Bubble Sort — सरल दृष्टिकोण

> अगर आसपास के तत्व गलत क्रम में हैं तो बार-बार उन्हें swap करें।

**प्रदर्शन** `[5, 3, 8, 1]` के साथ:

| पास | तुलनाएँ | परिणाम |
|------|---------|--------|
| 1 | 5→3 swap, 5→8 छोड़ें, 8→1 swap | [3, 5, 1, **8**] |
| 2 | 3→5 छोड़ें, 5→1 swap | [3, 1, **5**, 8] |
| 3 | 3→1 swap | [**1**, 3, 5, 8] |

**समय: O(n²) | Space: O(1)**

<Callout type="info">
Bubble sort केवल सीखने के लिए उपयोगी है। Interview में, इसका उल्लेख करें यह दिखाने के लिए कि आप जानते हैं कि बेहतर algorithms क्यों मौजूद हैं — फिर तुरंत merge sort या quick sort पर जाएँ।
</Callout>

## Merge Sort — विभाजित करो और जीतो

> Array को बार-बार दो हिस्सों में विभाजित करें, फिर sorted हिस्सों को वापस merge करें।

**3 चरणों में algorithm:**

1. **विभाजन** — array को दो हिस्सों में बाँटें
2. **विजय** — हर हिस्से को पुनरावर्ती रूप से sort करें
3. **Merge** — दो sorted हिस्सों को एक sorted array में मिलाएँ

**Merge चरण** सबसे महत्वपूर्ण है — हर हिस्से के सामने के तत्वों की तुलना करें:

| बायाँ | दायाँ | चयन | अब तक का परिणाम |
|-------|-------|------|------------------|
| **3**, 27, 38 | **9**, 43, 82 | 3 | [3] |
| 27, 38 | **9**, 43, 82 | 9 | [3, 9] |
| **27**, 38 | 43, 82 | 27 | [3, 9, 27] |
| 38 | 43, 82 | 38 | [3, 9, 27, 38] |
| — | 43, 82 | 43, 82 | [3, 9, 27, 38, 43, 82] |

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

**समय: O(n log n) | Space: O(n)**

<ThinkAboutIt>
O(n log n) क्यों? Array log(n) बार विभाजित होता है (आधा-आधा)। हर स्तर पर, हम O(n) merge कार्य करते हैं। कुल: n कार्य × log(n) स्तर = O(n log n)।
</ThinkAboutIt>

## Sorting Algorithm तुलना

| Algorithm | सर्वश्रेष्ठ | औसत | सबसे खराब | Space | Stable? |
|-----------|------------|------|-----------|-------|---------|
| Bubble Sort | O(n) | O(n²) | O(n²) | O(1) | हाँ |
| Merge Sort | O(n log n) | O(n log n) | O(n log n) | O(n) | हाँ |
| Quick Sort | O(n log n) | O(n log n) | O(n²) | O(log n) | नहीं |
| Heap Sort | O(n log n) | O(n log n) | O(n log n) | O(1) | नहीं |

<Callout type="info">
**Stable** का मतलब है कि समान तत्व अपना मूल क्रम बनाए रखते हैं। यह तब महत्वपूर्ण है जब कई keys से sort किया जाए — जैसे, पहले नाम से sort करें, फिर उम्र से। एक stable sort समान उम्र के groups में नाम का क्रम बनाए रखता है।
</Callout>

## Binary Search — समस्या को आधा करना

> एक **sorted** array में target खोजें, हर चरण में आधे तत्वों को हटाकर।

**पूर्वापेक्षा:** Array sorted होना चाहिए।

**प्रदर्शन** — `[2, 5, 11, 15, 23, 37, 41, 56]` में `23` खोजें:

| चरण | रेंज | मध्य | तुलना | क्रिया |
|------|------|------|-------|--------|
| 1 | [2..56] | 15 | 23 > 15 | दाएँ हिस्से में खोजें |
| 2 | [23..56] | 37 | 23 &lt; 37 | बाएँ हिस्से में खोजें |
| 3 | [23] | 23 | 23 = 23 | **index 4 पर मिला!** |

**3 चरण** 8 (linear scan) के बजाय। 10 लाख तत्वों के लिए: **20 चरण** 10,00,000 के बजाय।

```python
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2  # avoids overflow
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1  # not found
```

**समय: O(log n) | Space: O(1)**

<FunFact>
Binary search को सही तरीके से implement करना भ्रामक रूप से कठिन है। Jon Bentley के एक अध्ययन में पाया गया कि **90% पेशेवर programmers** bug-free binary search नहीं लिख सकते। सबसे आम bug? `(left + right) / 2` में integer overflow — इसीलिए हम `left + (right - left) // 2` का उपयोग करते हैं।
</FunFact>

## Binary Search के प्रकार

Binary search केवल "सटीक मान खोजो" तक सीमित नहीं है। ये प्रकार लगातार सामने आते हैं:

| प्रकार | विवरण | मुख्य बदलाव |
|--------|--------|------------|
| **Lower bound** | पहली position जहाँ तत्व डाला जा सकता है | `left` जब `nums[mid] >= target` → `right = mid` |
| **Upper bound** | आखिरी occurrence के बाद की position | `left` जब `nums[mid] > target` → `right = mid` |
| **Rotated array में खोज** | अज्ञात pivot पर rotate किया गया sorted array | पहले जाँचें कि कौन सा हिस्सा sorted है |
| **Answer space में खोज** | किसी शर्त को पूरा करने वाला min/max खोजें | Array पर नहीं, उत्तर पर binary search |

<ThinkAboutIt>
"Answer space पर binary search" एक शक्तिशाली तकनीक है। उदाहरण: "D दिनों में पैकेज भेजने के लिए न्यूनतम क्षमता क्या है?" न्यूनतम और अधिकतम संभव क्षमता के बीच binary search करें, यह जाँचते हुए कि हर उम्मीदवार काम करता है या नहीं। यह एक optimization समस्या को decision समस्या में बदल देता है।
</ThinkAboutIt>

## AI से जुड़ाव: Ranking और Retrieval

Sorting और searching AI सिस्टम के परिणाम देने के तरीके के लिए मूलभूत हैं:

**Search engines (Google, Bing):**

- **Inverted indexes** — हर keyword के लिए document IDs की sorted सूचियाँ
- इन indexes पर **binary search** sub-millisecond lookups के लिए
- कई keywords के परिणाम मिलाने के लिए **merge operations**

**Recommendation systems:**

- उम्मीदवारों को predicted score के अनुसार sort करें (relevance ranking)
- **Top-K selection** — पूरी तरह sort किए बिना K सर्वश्रेष्ठ items खोजें (partial sort)
- **Approximate Nearest Neighbour** search binary search जैसी tree-based संरचनाओं का उपयोग करता है

**Database query optimisation:**

- B-trees (binary search trees का सामान्यीकरण) हर SQL index को शक्ति प्रदान करते हैं
- Query planners sequential scan (O(n)) और index scan (O(log n)) के बीच चुनते हैं

<FunFact>
Google प्रतिदिन 8.5 अरब से अधिक searches प्रोसेस करता है। हर query सैकड़ों अरब web pages के index में खोज करती है। Sorted indexes और binary search के बिना, एक ही query में milliseconds के बजाय मिनट लगेंगे।
</FunFact>

## Complexity चीट शीट

| ऑपरेशन | Unsorted Array | Sorted Array |
|---------|---------------|-------------|
| खोज | O(n) | O(log n) |
| Insert | O(1) | O(n)* |
| Min/max खोजें | O(n) | O(1) |
| k-वाँ तत्व खोजें | O(n) | O(1) |

*sorted order बनाए रखने के लिए तत्वों को shift करना होगा

<Callout type="info">
समझौता स्पष्ट है: **sorting की शुरुआती लागत O(n log n) है**, लेकिन यह हमेशा के लिए O(log n) searches को सक्षम बनाता है। यदि आप O(log n) से अधिक बार search करते हैं, तो sorting अपनी लागत वसूल कर लेता है।
</Callout>
