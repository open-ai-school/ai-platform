# Strings & Pattern Matching

हर बार जब आप कोई search query टाइप करते हैं, मैसेज भेजते हैं, या AI से सवाल पूछते हैं — strings प्रोसेस हो रही होती हैं। इनकी आंतरिक कार्यप्रणाली समझना interview में सफलता और AI की समझ दोनों को अनलॉक करता है।

## Strings की आंतरिक संरचना

एक string बस एक **character array** है जिसमें विशेष नियम हैं।

<Illustration src="/images/lessons/craft/strings-patterns.svg" alt="String as character array, palindrome two-pointer check, and sliding window technique" caption="Strings character arrays हैं (ऊपर-बाएँ)। Palindromes की जाँच दोनों सिरों से दो pointers से होती है (ऊपर-दाएँ)। Sliding window इष्टतम substrings खोजता है (नीचे)।" />

**मुख्य तथ्य:**

- **Immutable** अधिकांश भाषाओं में (Java, Python, Go) — हर "बदलाव" एक नया string बनाता है
- **Indexed** — किसी भी character को O(1) में एक्सेस करें
- **Encoded** — UTF-8, UTF-16, ASCII यह प्रभावित करते हैं कि characters bytes से कैसे मैप होते हैं

<Callout type="info">
Python में, `s = "hello"` फिर `s[0] = "H"` एक error देता है। Strings **immutable** हैं। संशोधित करने के लिए `s = "H" + s[1:]` या `list(s)` का उपयोग करें।
</Callout>

## Pattern 1: String Reversal

> दो pointers का उपयोग करके एक string को in-place उलटें।

**विज़ुअल प्रदर्शन** — `"HELLO"`:

| चरण | L ptr | R ptr | क्रिया | परिणाम |
|------|-------|-------|--------|--------|
| 1 | H (0) | O (4) | Swap | **O**ELL**H** |
| 2 | E (1) | L (3) | Swap | O**L**L**E**H |
| 3 | L=L | — | पूर्ण | OLLEH |

```python
def reverse_string(s: list[str]) -> None:
    left, right = 0, len(s) - 1
    while left < right:
        s[left], s[right] = s[right], s[left]
        left += 1
        right -= 1
```

**समय: O(n) | Space: O(1)** — कोई अतिरिक्त मेमोरी आवश्यक नहीं।

## Pattern 2: Palindrome जाँच

> क्या string आगे और पीछे से एक जैसा है?

**Two-pointer दृष्टिकोण** — दोनों सिरों से तुलना करें, अंदर की ओर बढ़ें:

1. `left = 0`, `right = len - 1` सेट करें
2. अगर `s[left] != s[right]` → **palindrome नहीं है**
3. दोनों pointers को अंदर की ओर ले जाएँ
4. अगर वे मिल जाएँ → **palindrome की पुष्टि**

```python
def is_palindrome(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True
```

<ThinkAboutIt>
"A man, a plan, a canal: Panama" को आप कैसे handle करेंगे? आपको **non-alphanumeric characters को छोड़ना** होगा और **case को अनदेखा** करना होगा। हर तुलना से पहले `while left &lt; right and not s[left].isalnum(): left += 1` जोड़ें।
</ThinkAboutIt>

## Pattern 3: Anagram Detection

> क्या दो strings anagrams हैं? (वही characters, अलग क्रम)

**Hash map दृष्टिकोण** — character frequencies गिनें:

```python
def is_anagram(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False
    count = {}
    for c in s:
        count[c] = count.get(c, 0) + 1
    for c in t:
        count[c] = count.get(c, 0) - 1
        if count[c] < 0:
            return False
    return True
```

| | "listen" | "silent" |
|---|----------|----------|
| l | 1 | 1 |
| i | 1 | 1 |
| s | 1 | 1 |
| t | 1 | 1 |
| e | 1 | 1 |
| n | 1 | 1 |

सभी counts शून्य → **anagram की पुष्टि**।

## Pattern 4: Sliding Window

> बिना दोहराए characters वाली सबसे लंबी substring खोजें।

**तकनीक:** वैध characters की एक "window" बनाए रखें। दाएँ विस्तार करें, duplicate मिलने पर बाएँ से सिकोड़ें।

**प्रदर्शन** `"ABCABCBB"` के साथ:

| चरण | Window | देखे गए | लंबाई |
|------|--------|---------|-------|
| 1 | **A** | A | 1 |
| 2 | **AB** | A, B | 2 |
| 3 | **ABC** | A, B, C | 3 |
| 4 | A duplicate है → सिकोड़ें → **BCA** | B, C, A | 3 |
| 5 | B duplicate है → सिकोड़ें → **CAB** | C, A, B | 3 |

**उत्तर: 3** ("ABC", "BCA", या "CAB" में से कोई भी)

```python
def length_of_longest_substring(s: str) -> int:
    seen = {}
    left = max_len = 0
    for right, char in enumerate(s):
        if char in seen and seen[char] >= left:
            left = seen[char] + 1
        seen[char] = right
        max_len = max(max_len, right - left + 1)
    return max_len
```

<Callout type="info">
Sliding window pattern **दर्जनों** substring समस्याओं को हल करता है। मुख्य अंतर्दृष्टि: अगर window का विस्तार किसी शर्त को तोड़ता है, तो बाएँ से सिकोड़ें जब तक शर्त फिर से पूरी न हो।
</Callout>

## AI से जुड़ाव: Tokenization

किसी भी NLP model (GPT, BERT, LLaMA) के टेक्स्ट प्रोसेस करने से पहले, वह **tokenize** करता है — ऊपर दिए patterns पर आधारित तकनीकों का उपयोग करके टेक्स्ट को sub-word टुकड़ों में विभाजित करता है।

**BPE (Byte Pair Encoding) कैसे काम करता है:**

1. अलग-अलग characters से शुरू करें: `["h", "e", "l", "l", "o"]`
2. आसन्न pairs गिनें — सबसे ज़्यादा बार आने वाला pair: `("l", "l")`
3. Merge करें: `["h", "e", "ll", "o"]`
4. वांछित vocabulary size तक दोहराएँ

**यह इसका उपयोग करता है:**

- **Frequency counting** (hash maps) आम pairs खोजने के लिए
- **Sliding window** आसन्न characters को scan करने के लिए
- **String manipulation** tokens को merge करने के लिए

<FunFact>
GPT-4 के tokenizer में लगभग 1,00,000 tokens हैं। शब्द "indescribable" शायद ["in", "desc", "rib", "able"] बन जाए — 4 tokens। "the" जैसे आम शब्द एक ही token हैं। इसीलिए AI models शब्दों में नहीं, बल्कि tokens में "सोचते" हैं!
</FunFact>

## Complexity सारांश

| Pattern | समय | Space | मुख्य तकनीक |
|---------|------|-------|-------------|
| String reversal | O(n) | O(1) | Two pointers |
| Palindrome check | O(n) | O(1) | Two pointers |
| Anagram detection | O(n) | O(1)* | Hash map counting |
| Sliding window | O(n) | O(k) | Hash set + two pointers |

*O(1) क्योंकि character set निश्चित है (26 अक्षर)

<ThinkAboutIt>
Sliding window O(n) क्यों है और O(n²) नहीं? क्योंकि हर character को window में **अधिकतम एक बार** जोड़ा और हटाया जाता है। बायाँ pointer केवल आगे बढ़ता है, कभी पीछे नहीं — इसलिए सभी चरणों में कुल कार्य linear है।
</ThinkAboutIt>
